# Consulting FAQ Builder

## Steps
1. Write a clear, canonical answer to `knowledge/faqs/` (one file per question).
2. Link any related insights, content, or case studies.
3. Flag it as a content candidate if it has broad relevance (→ `content/ideas/`).
4. Note where it can be reused (sales calls, proposals, onboarding).

Principle: never answer the same question twice. Source: Ch. 4.
